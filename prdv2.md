## 0. 核心领域逻辑与数据定义 (Domain Rules)
在定义页面交互前，确立系统的核心指标与算法基准。

* **CI (清醒指数, Clear Index)**
    * **初始值**: 1.0。
    * **动态调整**: 单次大额“花钱/出力”未得回馈，或经历“单方面长等待”，CI -0.1；收到正向反馈（高 IQS），CI +0.05。
    * **自然衰减**: 连续 3 天无任何记录产生，系统判定情绪进入冷静期，CI 自动恢复 0.1（最高不超过 1.0）。
* **TI (沉没成本, Total Investment)**
    * **公式**: `TI = Σ 花钱金额 + Σ (出力时长 * (hourlyRate * 分类权重))`。回馈记账不产生负数 TI 。
* **IQS (互动质量分, Interaction Quality Score)**
    * 用于量化对方回复的质量。基于两维度打分：态度（冷暴力 -10，敷衍 -5，正常 0，主动 +5）+ 媒介（文本 1，语音 2，图片/视频 3）。
* **hourlyRate (时薪基准)**
    * 通过初始化配置获取，代表用户的自我时间估值（元/小时），作为“出力”转换为经济成本的基准乘数。

## 二、核心页面级详细设计

### 2.1 首页：顶部清醒指数区
* **UI结构**:
    * [标题：清醒指数 + (i) 提示图标]
    * [CI 数值] (大字 48px) 
    * [状态标签] (小字: 健康/警告/危险) 
    * [进度条(红绿渐变)] 
* **认知补足（低成本解释路径）**:
    * 点击 (i) 图标，通过原生 Tooltip 或轻量 Bottom Sheet 展示说明：
        * “什么是清醒指数 (CI)?
满分为 1.0。它衡量你的‘上头’与‘倒贴’程度。数值越低，说明你在这段关系里单方面投入的沉没成本（金钱、精力、等待时间）越高。低于 0.3 时，系统将拉响危险警报。”
* **文案与规则**: 数值保留2位小数 。
    * `≥1`: 健康，“你是清醒的” 
    * `0.3~1`: 警告，“开始不对劲了” 
    * `<0.3`: 危险，“你在倒贴人生” 

### 2.2 首页：等价物冲击区
**定位升级**：该卡片为基于净投入的双态组件，避免在“对方回馈良好”时出现“暴击”语义冲突。

* **净投入定义**: `净投入 = 总花钱 + 出力折算 - 总回馈`。
* **状态机**:
    * **状态A：亏损期 (净投入 > 0)**:
        * **模块名称**: 等价物暴击
        * **UI语境**: 维持“失去感”与警示氛围。
        * **文案**: “这些钱足够你买 [X]，而你只收到了 Ta 的 [最近一次冷反馈]。”
    * **状态B：盈余期/健康期 (净投入 ≤ 0)**:
        * **模块名称**: 情绪盈余（或 价值满足）
        * **UI语境**: 数字与强调色从警示色切换为温暖激励色，强调“旗鼓相当”。
        * **示例文案**: “Ta 近期的回馈，价值已经相当于送了你 [1台 Switch]。旗鼓相当的投入才是最健康的。”

* **动态等价物库**: 根据用户在 Onboarding 阶段选择的偏好标签（如：数码、美妆、游戏），动态替换等价物映射表。
* **UI 与文案规则**: 最多展示2个等价物 。应用场景化沉浸式文案：
    * *公式*: `这些钱够你买 {等价物数量与名称}，{获得感/爽感描述}。而你只收到了 {最近一次负反馈}`
    * *示例*: “这些钱够你买 4台 Switch，足够你和兄弟通宵爽玩一整年。而你只收到了 Ta 的 12句嗯”

### 2.3 首页：趋势图

* **交互**: 点击某天高亮，长按弹出 Tooltip 。Tooltip 内容包含日期、花钱、出力、回馈明细 。

### 2.4 首页：AI点评模块

* **触发与文案类型**:
    * CI下降连续3天: 嘲讽 
    * CI稳定: 中性 
    * CI上升: 鼓励 

### 2.5 快捷操作区

## 三、记一笔 (Bottom Sheet 弹窗)

顶部支持 [投入-花钱] [投入-出力] [回馈] 模式切换，切换时表单重置。采用级联选择器（如 Choice Chips）实现二级精细化分类。

### 3.1 投入：花钱模式
* **字段**:
    * 金额 (数字，必填) 
    * 分类 (二级级联):
        * 礼物: 首饰包包 / 数码外设 / 鲜花手工
        * 约会: 高档餐饮 / 电影演出 / 密室桌游
        * 转账: 清空购物车 / 节日红包 / 帮还账单
    * 备注 (文本，20字) 
* **逻辑**: 保存写入 Transaction (type=0, sub_type=expense_money)，关闭弹窗并刷新 Dashboard。

### 3.2 投入：出力模式 (重点)
* **UI**: 滑块选择 `0.5h ~ 24h`。
* **字段与分类 (二级权重调优)**:
    * 情绪价值 (权重 1.5x): 深夜树洞安慰 / 吵架主动破冰 / 精心准备惊喜
    * 体力劳动 (权重 1.0x): 跑腿接送 / 搬家打扫 / 排队代买
    * 时间沉没 (权重 0.8x): 单方面漫长等待 / 陪做不感兴趣的事
* **实时计算**: 每100ms 刷新 UI。`金额 = duration * (hourlyRate * 分类权重)`。
* **逻辑**: 保存写入 Transaction (type=0, sub_type=expense_effort)。

### 3.3 回馈模式 (重点冲销机制)

**定位**：记录对方提供的物质、情绪或行动价值，作为“收入项”用于扣减自身的总投入 (TI) 并提升清醒指数 (CI)。

* **UI**: 
    * 顶部切换至 [回馈] 模式。
    * **价值输入**：根据分类动态切换“直接金额输入”或“时长滑动条”。
    * **分类（二级级联选择器）**。
    * **备注（文本，20字）**。

#### 字段与分类字典库建议

* **物质回馈 (Material Return - 1:1 价值记账)**
    * **字段**: 直接输入金额/估值。
    * **二级分类**:
        * 收到礼物: 数码产品 / 首饰包包 / 鲜花手工
        * 对方买单: 请客吃饭 / 娱乐消费 / 旅行支出
        * 资金转账: 节日红包 / 报销代付 / 意外转账
* **情绪回馈 (Emotional Return - 系统按基准换算价值)**
    * **字段**: 时长滑动条 (0.5h ~ 24h)。
    * **二级分类 (权重 1.2x)**:
        * 走心沟通: 深度交流解决问题 / 积极回应诉求
        * 情绪支持: 低谷期倾听安抚 / 压力分担
        * 制造惊喜: 非节日浪漫举动 / 意外的仪式感
* **行动回馈 (Action Return - 系统按基准换算价值)**
    * **字段**: 时长滑动条 (0.5h ~ 24h)。
    * **二级分类 (权重 1.0x)**:
        * 分担任务: 家务协助 / 代办事项 / 跑腿照顾
        * 专属陪伴: 陪做我不感兴趣的事 / 远距离奔波探望

#### 领域规则与逻辑闭环

* **非物质回馈估值**: 当选择“情绪/行动回馈”时，系统反向计算对方价值：`冲销金额 = duration * (对方的 hourlyRate估值 * 分类权重)`。
* **TI 冲销底线保护**: 写入 Transaction (type=1, 回馈) 后，系统重新计算总 TI（投入）。若 `总投入 - 总回馈 < 0`，逻辑层需确保展示的净投入最小为 0，防止数据“刷成负数”导致心理失衡。
* **CI 联动**: 发生有效回馈记录时，触发 Domain Event 更新清醒指数（如 CI + 0.1，最高不超过 1.0），代表对方有正向反馈，关系健康度提升。
* **保存行为**: 保存并关闭弹窗，刷新 Dashboard。在趋势图中，回馈项作为“负向柱状体”或对比色色块展示，直观呈现收支对冲效果。

## 四、等待计时器 (包含持久化容灾机制)

### 4.1 FSM 状态机与水合机制 (Rehydration)
* **生命周期**: `Idle -> Running -> Evaluating -> Idle` 。
* **异常处理**: 状态变更为 `Running` 时，将 `startTime` 时间戳同步写入本地存储。若 App 进程被系统回收，冷启动时读取 `startTime` 重建状态机并补齐时间差，确保计时不断流。

### 4.2 计时器 UI 与结算
* **UI 显示**: `00:32:15` 倒计时 与 状态文案 。
* **颜色规则**: `<1h` 绿，`1~4h` 黄，`>4h` 红 。
* **手动结算 [终于回了]**: 弹出选择器评估 Ta 的态度 (敷衍/正常/主动/冷暴力) 与媒介 (文本/语音/图片) 。计算 IQS 并刷新 CI 。
* **自动判定**: 当 `等待 > 阈值 + 30分钟` 时，自动写入 Interaction (base=-1) 并生成 Transaction (等待成本) 。

### 一、 首页核心视觉：状态大圆盘 (Status Circle)

### 1.1 视觉定位
圆盘占据首页约 1/3 的黄金视觉区，是用户感性情绪的集中承载点。它必须与顶部理性的 CI 指数仪表盘在空间上彻底剥离。

### 1.2 状态机详细定义 (FSM Implementation)

#### 状态 A: Idle (未计时/初始态)
* **UI 表现**: 
    * 静态深色圆圈（建议带细微的描边质感）。
    * 圆心显示：默认文案“开启漫长等待”或一个具有暗示性的 Icon。
    * [cite_start]底部按钮：在圆盘下方或圆盘内部底端，显示操作入口 `[开始熬]`。 [cite: 142]
* **触发动作**: 点击按钮 -> 写入当前时间戳 `startTime` 至本地持久化存储 -> 切换至 **Running** 态。

#### 状态 B: Running (计时中/焦虑态)
* **UI 表现**: 
    * **呼吸灯动效**: 圆环开启周期的亮度/阴影变化，模拟人的呼吸感。
    * [cite_start]**核心计时**: 圆心显示巨大的跳动时间戳（格式 `HH:mm:ss`）。 [cite: 147]
    * **颜色梯度**: 
        * [cite_start]<1h: 极简绿（代表耐心尚存）。 [cite: 153]
        * [cite_start]1~4h: 警告黄（代表开始烦躁）。 [cite: 153]
        * [cite_start]>4h: 警报红（代表尊严危险）。 [cite: 153]
    * [cite_start]**动态挂件**: 圆盘下方悬浮出现一个可点击的胶囊按钮：`[回了么？]`。 [cite: 154]
* [cite_start]**持久化逻辑**: 在 `build` 方法中需通过 `startTime` 自动恢复计时，确保 App 后台重启后倒计时不断流。 [cite: 139]

#### 状态 C: Evaluating (结算中/评估态)
* **触发动作**: 点击“回了么？” -> 暂停计时。
* **UI 表现**: 圆盘区域翻转或弹出一层轻量级选择器。
* [cite_start]**评估字段**: [cite: 156-164]
    * **态度评估**: 敷衍 / 正常 / 主动 / 冷暴力。
    * **媒介评估**: 文本 / 语音 / 图片。
* [cite_start]**执行逻辑**: 点击“确认” -> 计算 IQS 分数 -> 异步刷新顶部 CI 指数 -> 回归 **Idle** 态。 [cite: 165-169]

### 1.3 自动判定逻辑
* [cite_start]**超时逻辑**: 若 `Running` 状态持续超过 `阈值 + 30分钟` 且用户未主动点击“回了么？”，系统自动结算。 [cite: 172]

### 增量更新说明 (核心数据模型与 UI 规范)

#### 1. 领域模型：分类枚举树 (Domain Enums)
重构 `Transaction` 的分类模型，定义明确的上下级与属性权重：

* **顶级方向 (`TransactionDirection`)**: 
  * `EXPENSE` (支出/投入)
  * `RETURN` (回血/回馈)
* **支出子类 (`ExpenseCategory`)**: 
  * `FINANCIAL` (财务开销)
  * `EFFORT` (行动付出)
  * `TIME_FRICTION` (时间磨损)
  * `EMOTIONAL_DRAIN` (情绪消耗)
  * `OTHER` (其他)
* **回馈子类 (`ReturnCategory`)**: 
  * `MATERIAL` (物质回血, 冲销杠杆 1:1)
  * `INTIMACY` (亲密接触, 极高冲销杠杆，如 BaseValue = 1000)
  * `EMOTIONAL_VALUE` (情绪甜头, 中高冲销杠杆，如 BaseValue = 300)
  * `OTHER` (其他)

#### 2. 记账弹窗交互 (Bottom Sheet UI)
* **布局**: 顶部保留宽大的 `[投入]` 与较窄的 `[回血]` Tab。二级分类采用 `Wrap` 组件呈现的小 Chip 按钮。
* **“其他”分类约束**: 当选中 `OTHER` 类型的 Chip 时，下方 `remark` (备注) 输入框变为**强必填项**，Placeholder 提示：`“一句话描述你干了啥/得到了啥”`。
* **记账暴击拦截 (Sarcastic Toast)**: 
  * 触发条件：当用户在 `[回血]` 中提交了一笔 `INTIMACY` (如：主动牵手) 时。
  * 交互：触发 `HapticFeedback.heavyImpact()`，随后弹出黑底红字的 `AlertDialog`。
  * 文案预设：“⚠️ 账本预警：检测到 1 次零成本肢体接触。系统已为您强行冲销 ¥1000 沉没成本。看，只要一点甜头，你的底线就能无限退让。已强制入账。”

#### 3. 复盘时间轴渲染层 (Timeline UI)
* **“其他”备注提权**: 
  * 若 `category == OTHER`，时间轴的主标题直接绑定 `transaction.remark`，不显示“其他”二字。
* **情绪半衰期状态机 (Decay State)**:
  * 具有高杠杆的回馈卡片具备两种 UI 状态：`active` (暧昧发光) 和 `decayed` (褪色破裂)。
  * **Active 状态**: 刚记账的 24 小时内，卡片带有粉色/暖金色的微光外发光效果 (`BoxShadow`)。
  * **Decayed 状态**: 超过时效，或在此期间又新增了 `TIME_FRICTION` 记录时，卡片失去光泽，颜色变为灰暗，并在备注下方强行追加一条划线警告：`[药效已过，滤镜破碎，账本赤字已回滚]`。
* **数值波动高亮**: 非物质维度的进出，必须在卡片右侧直观显示等效影响，如 `📉 CI -0.05`。
