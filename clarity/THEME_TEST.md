## 🎯 建议测试流程

1. **从 Level 1 开始**（场景 4）- 建立健康状态的视觉基准
2. **切换到 Level 2**（场景 5）- 观察警告状态的 UI 变化
3. **切换到 Level 3**（场景 6）- 体验危险状态的视觉冲击

每次切换后记录：
- 色彩过渡是否自然？
- 警告提示是否醒目？
- 数据展示是否清晰？
- 整体氛围是否符合预期？

---

**注意**：目前动态主题的 UI 变化可能还未完全实现（如红色闪烁、按钮变大等），这些数据主要用于验证**核心指标计算**和**评级系统**是否正确。UI 层面的主题切换需要在 `app_theme.dart` 和各个 Widget 中进一步实现。
