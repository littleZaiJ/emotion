# 产品需求文档 (PRD)：恋爱账单 (Project Clarity)

**文档控制版本**: V 1.0 (开发定稿版)  
**产品线**: C端移动应用 (Flutter)  
**核心定位**: 恋爱沉没成本审计与决策支持系统  
**关键依赖**: 本地数据库 (Isar/SQLite)

---

## 一、 核心算法与指标定义 (Core Algorithms)

本应用的核心是计算用户的“**清醒指数 (Clarity Index, CI)**”，该指数直接驱动全局 UI 的动态变化。

### 1.1 资产投入计算 (Total Investment, TI)

记录用户付出的总成本（转化为法币单位：元）。

**公式**：  
`TI = Σ 金钱支出 + Σ (劳务耗时(h) × 用户基准时薪) - Σ 收到回礼金额`

**约束**：  
- TI 最低为 0（若回礼大于支出，计为 0）。

---

### 1.2 互动质量评分 (Interaction Quality Score, IQS)

单次聊天的回报质量，用于替代单纯的“字数”。

**公式**：  
`IQS = (基础态度分 × 媒介乘数) - 延迟惩罚分`

#### 变量定义：

- **基础态度分 (Base Score)**:
  - 敷衍/毫无营养 (哦/嗯/单表情) = `0.5`
  - 正常就事论事 = `1.0`
  - 主动分享/开启话题 = `2.0`
  - 情绪反扑/冷暴力 = `-1.0`

- **媒介乘数 (Medium Multiplier)**:
  - 纯文本 = `1.0`
  - 长语音 (>10s) = `1.2`
  - 图片/视频/链接 = `1.5`

- **延迟惩罚分 (Delay Penalty)**:
  - `0-1小时`：`0`
  - `>1小时`：`(等待分钟数 - 60) × 0.01`，**最大惩罚上限为 2.0**

---

### 1.3 清醒指数 (Clarity Index, CI)

反映关系的健康度。

**公式**：  
`CI = (Σ IQS × 转换系数K) / (TI + 1)`

**说明**：  
- `K` 为常数（如 `100`），用于将 IQS 的小分值拉平至与金钱成本可对比的量级；
- 分母 `+1` 防止除零错误。

---

## 二、 动态主题系统 (Dynamic Theme State Machine)

全局 UI 风格由最近 7 天的 CI (清醒指数) 严格控制。

| 状态枚举 (State) | 触发阈值 | 视觉规范 (Visual & UI) | 交互限制与变化 |
|------------------|----------|------------------------|----------------|
| **ThemeState.Safe** | `CI ≥ 1.0` | 主色：`#00E676` (极光绿)<br>图表：平滑贝塞尔曲线<br>字体：正常粗细 | 无限制<br>主打“稳健记录” |
| **ThemeState.Warning** | `0.3 ≤ CI < 1.0` | 主色：`#FFC107` (警告黄)<br>图表：出现折角<br>顶部常驻 Warning 提示栏 | 记账按钮触发轻微震动 (Haptic Feedback) |
| **ThemeState.Danger** | `CI < 0.3` | 主色：`#FF2D55` (鲜红) + `#000000` (纯黑底)<br>图表：尖锐锯齿<br>关键按钮心跳缩放动效 | 【记一笔】按钮缩小 20%<br>新增常驻大按钮【发布求助】 |

## 三、 核心业务模块详述 (Functional Specifications)

### 3.0 底部导航 (Bottom Navigation)

底部 5 位布局（中间为悬浮 FAB）：
`[ 大盘 ] [ 复盘 ] [ +记一笔 ] [ 体检单 ] [ 设置 ]`

### 3.1 模块 A：记账录入 (Input System)

- **入口**: 首页底部 FAB (Floating Action Button)
- **交互**: 底部弹窗 (Bottom Sheet)
- **倒贴方向**：顶部使用 SegmentedButton 切换【花钱】/【出力】
- **回馈方向**：仅录入“对方给你的回馈金额”（不提供出力）

#### 规则说明：

**倒贴 / 花钱模式**：
- 必填项：金额 (最多两位小数)
- 选填项：分类 (单选：红包/礼物/餐饮/代付)、备注 (Max: 20字)

**倒贴 / 出力模式 (劳务)**：
- 必填项：耗时 (滑动选择器：0.5h 步进，最大 24h)
- 选填项：分类 (单选：跑腿/接送/工作/情绪劳动)

**回馈模式（Income）**：
- 必填项：回馈金额 (最多两位小数)
- 选填项：回馈类型 (单选：转账/红包、回礼、请吃饭、报销/AA)
- 选填项：快捷备注（单选或覆盖输入，Max: 20字）
  - 温和：道歉补偿 / 纪念日 / 主动约你 / 顺手回礼
  - 更激进：约会 / 牵手 / 亲嘴 / 上床

**联动逻辑**:  
滑动时长时，下方 Text 实时更新：  
`折现金额 = 耗时 × 本地存储的 hourlyRate`

**异常处理**:  
未填金额或耗时为 0 时，【保存】按钮置灰 (Disabled)。

---

### 3.2 模块 B：等待计时器 (Timer Widget & App UI)

- **入口**: 首页固定卡片区域 / 桌面 Widget

#### 状态机 (TimerState)：

| 状态 | 行为 |
|------|------|
| **Idle** | 显示“发送消息了？开始计时” |
| **Running** | 显示正计时 `HH:mm:ss`<br>当时间 > `UserSettings.dignityThreshold`，卡片背景变 `#FF2D55`，文案追加“尊严预警” |
| **Evaluating** | 点击“终于回了”进入该状态<br>弹出【质量评价选择器】（4个基础态度 + 3个媒介选项） |

**数据流**:  
用户在 Evaluating 确认后，生成一条 Interaction 记录写入数据库，自动重置为 Idle 并触发全局 CI 重新计算。

**自动判定（已读不回）**：
- 当进入“尊严预警”后，超过 `UserSettings.autoMarkAfterMin` 仍未确认 Evaluating，则自动结束本次等待并写入：
  - Transaction：一笔“等待成本”记账（note=已读不回）
  - Interaction：baseAttitude=-1.0，mediumType=Text，calculatedIQS 直接落库

---

### 3.3 模块 C：清醒仪表盘 (Dashboard)

#### 数据渲染要求：

- **等价物计算引擎**:  
  取当前 `TI` (总投入)，除以预设锚点价格：
  - 奶茶 ¥20
  - Switch ¥2000
  - iPhone ¥6000  
  向下取整展示最大的可用等价物。

- **文案渲染逻辑**:  
  `这些钱够你买 ${Count} ${ItemName}。而你只收到了 ${Total_IQS} 分的情绪回馈。`

- **K 线图 (fl_chart)**:  
  - X轴：近 7 天 (日期格式 `MM-dd`)
  - Y轴：每日 TI 总额
  - 交互：长按柱状图弹出 Tooltip 显示该日明细（花钱总额、出力总额）

---

### 3.4 模块 D：战况大厅 (Community MVP)

**发布规则**:  
用户本地 Transaction 记录 >= 5 条方可解锁发布权限。

**数据脱敏**:  
用户点击发布时，系统自动截取近 30 天的 `TI` 和 `CI` 趋势数组、等价物文案。  
**禁止用户上传自定义图片**，全部由系统生成标准化的《战况卡片》。

**投票限制**:  
针对单篇帖子，每位设备用户仅限投票 1 次（点击 🔴止损 或 🟢继续）。

## 四、 数据字典 (Data Schema)

基于 Isar/SQLite 的实体关系定义。

### Table: Transaction (资金与劳务记录)

| 字段名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `id` | `Int` | Yes | 主键，自增 |
| `recordTime` | `DateTime` | Yes | 记录发生的时间戳 |
| `type` | `Int (Enum)` | Yes | 0=花钱(Expense), 1=出力(Labor), 2=收到回礼(Income) |
| `amount` | `Double` | No | 花钱/回礼时的具体金额 |
| `durationHours` | `Double` | No | 出力时的耗时 (小时) |
| `snapshotRate` | `Double` | No | 记录出力时的时薪快照 (防止后期修改时薪导致历史数据突变) |
| `category` | `String` | No | 分类标签 (如："礼物", "跑腿") |
| `note` | `String` | No | 备注（Max: 20字，用于快捷备注/解释） |

---

### Table: Interaction (互动与等待记录)

| 字段名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `id` | `Int` | Yes | 主键，自增 |
| `startTime` | `DateTime` | Yes | 发送消息开始等待的时间 |
| `endTime` | `DateTime` | No | 收到回复的时间。若为空表示仍在等待或已放弃 |
| `baseAttitude` | `Double` | No | 态度打分 (0.5, 1.0, 2.0, -1.0) |
| `mediumType` | `Int (Enum)` | No | 媒介类型 (0=Text, 1=Voice, 2=Media) |
| `calculatedIQS` | `Double` | No | 落库时直接算好的 IQS 最终分，避免重复计算 |

---

### Table: UserSettings (本地配置单例)

| 字段名 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `hourlyRate` | `Double` | `50.0` | 用户设定的劳务折算时薪 |
| `dignityThreshold` | `Int` | `240` | 尊严阈值时长（分钟），控制变红预警 |
| `autoMarkAfterMin` | `Int` | `30` | 超过尊严阈值后，多久自动判定为“已读不回” |

---

## 五、 异常与边界条件处理 (Edge Cases)

1. **时薪修改的历史兼容**：  
   用户修改 `UserSettings.hourlyRate` 时，仅对未来的 Transaction 生效。  
   历史数据的价值通过记录时的 `snapshotRate` 固定，确保财务数据的幂等性。

2. **跨天计时的处理**：  
   如果 Interaction 跨越了 0 点，等待时长不受自然日影响，依然计算总分钟数。  
   但在仪表盘展示“每日 IQS”时，该笔得分归属于 `endTime` 所在的日期。

3. **极端异常值过滤**：  
   若用户恶意输入单笔金额 > 1,000,000 元，暂不限制落库，  
   但在计算等价物和图表 Y 轴极值时，**最高封顶处理**，防止 UI 图表渲染溢出。
